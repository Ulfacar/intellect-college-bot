# UI админки под приёмную комиссию колледжа

Дизайн адаптации существующей админки (`app/admin/router.py` + `app/admin/templates/*`)
под воронку `admission`. Механика (htmx-опрос, drag-and-drop, перехват, поиск, инбокс,
архив, аудит, FAQ-редактор) НЕ меняется — меняются словари/ярлыки/тексты и один
доменный блок карточки лида. Только текстовое описание — Codex реализует.

---

## 1. `app/admin/router.py` — константы

Все переименования делаются в одном месте, шаблоны подхватят сами:

```
FUNNELS = [("admission", "Абитуриенты")]
FUNNEL_LABELS = {"admission": "Приёмная"}
FAQ_TABS = FUNNELS + [("common", "Общие")]

BOARD_COLUMNS = [
    ("greeting",      "Новые лиды"),
    ("qualification", "Квалификация"),
    ("progress",      "Консультация"),
    ("office",        "Приглашён на тест"),
    ("manager",       "У менеджера"),
    ("silent",        "Молчат (на дожим)"),
    ("follow_up",     "Повторное касание"),
]

OUTCOMES = [("won", "🎓 Поступает"), ("office", "📝 Пришёл на тест"), ("lost", "❌ Слив")]
```

Ключи колонок сознательно НЕ меняются (см. 01-tech-design §4.2): на них завязаны
`STAGE_TO_COLUMN`, CSS `.col[data-col=...]`, `COLUMN_TO_STAGE` (drag-and-drop) и
`SILENT_EXCLUDED_COLUMNS`. Меняются только человеческие ярлыки.

`SCENARIO_LABELS` (тумблеры ботов в /admin/system): `{"admission": "Приёмная"}` — на все
3 номера один сценарий, различаются `manager_name`/названием бота из реестра `BOTS`.

## 2. `boards.html` — шапка и брендинг

- `<title>` и `document.title` в `_pollStats()`: «Frunze — диалоги» → «Intellect College —
  приёмная» (два места: тег title и JS-строка с счётчиком `(n) …`).
- Блок `.brand`: «Frunze / диалоги» → «Intellect College / приёмная комиссия».
- Вкладки воронок рендерятся из `FUNNELS` — после правки констант останется одна вкладка
  «Абитуриенты» рядом с «Инбокс». Разметку цикла не трогаем (если появится второй сценарий —
  вкладка добавится сама).
- Бейджи `.funnel-badge[data-funnel=...]` в CSS: заменить селекторы `visa/tours/tickets`
  на один `[data-funnel="admission"]` (синяя гамма `--accent-soft`).
- Цветовая тема: остаётся текущая (синий #2563EB — нейтрально подходит IT-колледжу).
  Опционально (не блокер): `--green` акцент заменить на фирменный цвет колледжа, когда
  Emir пришлёт брендбук — сейчас НЕ трогать.
- Плейсхолдер поиска «Поиск по имени или номеру…» — оставить.

## 3. `_board.html` — доска

Шаблон универсальный (колонки приходят из route) — правок почти нет:

- Ярлыки отправителей `{'client':'Клиент','bot':'Бот','manager':'Вы'}` — оставить
  («Бот» во внутренней админке допустим; клиенту персона не раскрывается).
- Метрики (`Всего диалогов / Ждут ответа / Требуют ответа / Молчат / …`) — без изменений.
- Бейдж «горячий» (`lead_temperature == 'hot'`) — семантика сохраняется: горячий =
  готов на тест/оформление.

## 4. Карточка лида (доска + чат): поля абитуриента

Сейчас `_conversation.html` рендерит `c.qualification` как сырые ключи (`name`, ...).
Для приёмной ввести словарь человеческих ярлыков полей (в `router.py`, использовать в
шаблоне через переданный map либо отфильтрованный список пар):

```
QUALIFICATION_LABELS = {
    "name": "Имя",
    "grade_base": "База (после класса)",
    "direction": "Направление",
    "visit_time": "Удобное время (тест)",
    "escalation_reason": "Причина эскалации",
}
```

- Блок «Что узнал бот» → переименовать в **«Карточка абитуриента»**; выводить поля в
  фиксированном порядке (Имя → База → Направление → остальное), ключи — через
  `QUALIFICATION_LABELS` (незнакомые ключи показывать как есть — не терять данные).
- `grade_base` показывать как «после 9 класса» / «после 11 класса» (форматировать в route).
- В строке тегов шапки чата (`c.funnel · c.stage`) выводить ярлык стадии по-русски —
  взять label из `BOARD_COLUMNS` через `STAGE_TO_COLUMN` (сейчас сырое `stage`).
- Мини-карточка на доске (`_board.html .card`): дополнительно к имени/телефону вывести
  строку «после {grade_base} · {direction}», когда поля собраны (одна строка, серым,
  под `.card-id`) — менеджер видит квалификацию без клика.

## 5. Чат с перехватом

Механика перехвата не меняется (это ядро правила «не врёт → человек подключается»):

- Кнопка «Перехватить (бот замолчит)» / «Вернуть боту», статус «Бот на паузе — диалог
  ведёт менеджер», hint «Отправка ответа перехватывает диалог — бот замолкает» —
  тексты оставить как есть.
- Блок «AI для менеджера» (ai_summary / manager_next_step / escalation_reason) —
  без изменений; `escalation_reason` теперь наполняется из `handoff_to_manager(reason)`
  (см. 01-tech-design §2.1) — проверить, что причина видна менеджеру.

## 6. Быстрые ответы приёмной (`branding.QUICK_REPLIES["admission"]`)

Кнопки под чатом, подставляются в поле ввода (механика `fillReply` не меняется).
Тексты — факты whitelist из 02-dialog-ux, на русском (менеджер при необходимости
правит руками; ky-вариант менеджер набирает сам — кыргызские кнопки удвоили бы ряд,
в MVP не делаем):

| Направления | У нас 8 направлений — IT: кибербезопасность, программная инженерия и ИИ, DevOps, веб-разработка, цифровые продукты; бизнес: бизнес-аналитика и финансы, маркетинг, менеджмент, графдизайн и UX/UI. Какое интересует? |
| Тест | Вступительный тест — математика и английский, 1,5 часа. Подтвердим вам дату и время записи 😊 |

## 7. FAQ-редактор (`faq.html`)

- Вкладки берутся из `FAQ_TABS` → «Абитуриенты» + «Общие» (правка констант, шаблон не трогаем).
- `core/faq.VALID_FUNNELS = {"admission"}` (см. 01-tech-design §7) — форма создания
  правила автоматически ограничится admission/common.
- Seed-правила — по 02-dialog-ux §4 (паттерны ru + ky обязательно).

## 8. Аналитика (`analytics.html`)

- Захардкоженный список исходов в шаблоне (строка ~96: `('won','✅ Оплатил'), ('office',
  '🏢 Дошёл в офис'), …`) заменить на ярлыки приёмной: `won` → «🎓 Поступает», `office` →
  «📝 Дошёл до теста», `manager` → «👤 У менеджера», `lost` → «❌ Слив», `in_progress` →
  «⏳ В работе». Лучше — рендерить из `OUTCOMES`+доп.статусов, переданных из route,
  чтобы ярлыки жили в одном месте.
- Остальные метрики (воронка стадий, конверсия) — универсальные, не трогаем.

## 9. Прочее

- `system.html` / `_bot_toggles.html`: подписи ботов возьмутся из реестра `BOTS` (T7 —
  3 номера колледжа) и `SCENARIO_LABELS`; шаблоны не менять.
- `login.html`: заголовок/бренд «Frunze» → «Intellect College» (если захардкожен).
- `_attention.html` (инбокс): `funnel_label` подхватит «Приёмная» из констант; прочее без правок.
- Grep-критерий приёмки UI: в `app/admin/` не остаётся строк «Frunze», «тур», «виза»,
  «билет» (кроме истории в комментариях — их тоже почистить).
