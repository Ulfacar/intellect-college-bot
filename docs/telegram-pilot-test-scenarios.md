# Telegram-пилот — тестовые сценарии

> Сценарии для ручной и авто-проверки бота в Telegram-песочнице. Колонки: **вход** (что пишет
> тестировщик), **ожид. ответ** (суть), **intent**, **lead_status**, **bot_phase**, **owner**
> (dialog_owner), **handoff** (нужна ли передача менеджеру), **LLM** (должен ли вызываться),
> **критерий успеха**. Реальные номера/ФИО не использовать — только тестовые данные.
>
> Сокращения статусов: new, in_progress(ip), info_sent, callback(cb), thinking(th), invited(inv),
> tested_thinking(tt), pre_contract(pc), contract, rejected(rj), invalid_number(iv).
> bot_phase: greeting(gr), qualification(q), consultation(cons), waiting(w), handoff(h).
